# write-outline

**글(에세이)·강의(PPT) "개요"를 집요한 인터뷰로 정련하는 Claude Code 스킬.**

`/deep-interview`가 "얕고 일찍 멈춘다"는 문제를 정조준해 고친 개요 설계 하네스다. 목적·의도·주제·의뢰서·자료를 받아 **파악 보드 → 핵심 메시지 관문 → 칸별 신호등 게이트 → 개요 → 분리 검토**로 한 편의 개요를 끝까지 정련한다. 산출물은 **확정 개요**이며, 그 뒤(문장 윤문·슬라이드 제작)는 다운스트림 스킬로 인계한다.

> 이 스킬은 **개요를 혼자 써 주지 않는다.** "틀린 길로 가는 것을 막는 가드레일"로 작동해, 사용자(저자)의 판단을 *집요하게* 끌어내 개요를 함께 짠다.

## 핵심 메커니즘

| 단계 | 하는 일 |
|------|---------|
| **파악 보드** | 자료를 정독해 ✅확정/🟡추정/⬜빈칸으로 분류 + **출처 라벨·신뢰 등급**(provenance) — 오독을 막는다 |
| **핵심 메시지 관문** | 강조점을 *선결*로 확정. **위계**(의뢰서 > 보고서) → **역의심**(자료 표제 ≠ 강조점) → **백지 예외**(자료 너머 한 문장) → **둘째 논지**(주 논지를 받치거나 반박) |
| **옵션우선** | 백지 질문 금지. 자료에서 뽑은 2~3개 후보를 먼저 주고 고르거나 고치게 |
| **칸별 신호등** | 🔴L1 / 🟡L2 / 🟢L3 루브릭. 모든 하드 칸 🟢이면 통과 (가짜 정밀 가중합 폐기) |
| **사례·근거 쌍** | 섹션마다 사례와 근거가 맞물려야 통과 |
| **분리 검토자** | 인터뷰어와 분리된 검토자가 체크리스트로 검수(자가채점 방지) |

두 프로파일을 지원한다 — **원고(에세이)** 와 **강의(PPT)**. 강의는 `⏱ 시간 예산`·`🪧 1슬라이드=1메시지` 고유 하드 게이트를 추가한다.

## 설치

```bash
git clone https://github.com/riedle03/write-outline.git ~/.claude/skills/write-outline
```

`SKILL.md`의 프런트매터 `name: write-outline`이 인식되면 끝. 어느 폴더에서나 동작한다.

## 사용

- `/write-outline` 입력 → "무엇을 쓰십니까?(에세이/강의)"부터 시작.
- 또는 자동 트리거: *"이 주제로 개요 잡아줘"*, *"의뢰서랑 자료 줄게 질문해줘"*, *"deep-interview로는 부족해 더 집요하게"*.

## 어떻게 만들었나

이 스킬은 **테스트로 검증**하며 지었다(`DESIGN.md` 전문 기록):

1. **백테스트** — 자료만 보고 코어를 돌려, 강조점이 자료의 *틀린 표제*에 끌려가는 결함(H1)을 발견 → 핵심 메시지 관문(위계·역의심·백지·둘째 논지)으로 봉합.
2. **엔드투엔드 테스트** — 실제 완성 원고를 정답지로 콜드 실행 → 근거 유형·둘째 논지·섹션 순서 처방을 도출.
3. **TDD 경화**(`superpowers:writing-skills`) — 압박 시나리오로 RED→GREEN→REFACTOR. 압박 구멍을 **5회 반복 검증**으로 닫음(대조군 3/3 덤프 vs 처방 0/5, 재검증 5/5 수렴).
