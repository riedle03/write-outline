# write-outline — 집필 전 개요 설계 인터뷰 하네스

글/강의 개요를 **집요한 인터뷰 → 파악 보드 → 핵심 메시지 관문 → 칸별 게이트 → 개요 → 검토**로 정련한다. `/deep-interview`가 "얕고 일찍 멈춘다"는 문제를 정조준해 고친 하네스다. 산출물은 **확정 개요**이며, 그 뒤(문장 윤문·구절 수정·슬라이드 제작)는 다운스트림 스킬로 인계한다.

> **자기완결형 글로벌 스킬.** 분리된 검토자는 `general-purpose` 에이전트에 인라인 프롬프트로 디스패치한다(프로젝트 로컬 에이전트에 의존하지 않는다). 프로파일별 상세(칸·루브릭·도전 항목·검토 체크리스트)는 `references/`에 둔다.

## 0. 진입 라우터 — 첫 질문 (반드시 먼저)

> **"무엇을 쓰십니까?"** — ⓐ 일반 원고(에세이) / ⓑ 강의용 PPT 원고
> (자유 입력 허용. 사용자가 이미 밝혔으면 묻지 않고 진행.)

- **ⓐ → 프로파일 A:** `references/profile-a-원고.md` 로드.
- **ⓑ → 프로파일 B:** `references/profile-b-강의.md` 로드.
- 코어(6원칙·옵션우선·루브릭·신호등·관문·검토)는 공통. **칸 세트·게이트·도전 항목만** 프로파일에서 갈아 끼운다.

## 1. 집요한-인터뷰 코어 6원칙 (매 라운드 강제)

1. **막연답 거부(ⓐ)** — 추상 답이면 점수 안 주고 좁히는 꼬리질문.
2. **3단 파고들기(ⓑ)** — "왜 → 구체적으로(숫자·고유명사·장면) → 그래서 뭐가 달라지나" 최소 3단.
3. **게이트 전 산출 금지(ⓒ)** — 모든 하드 칸 🟢 전엔 개요 안 쓴다. (즉석 생성 거부.)
4. **도전 모드 상시(ⓓ)** — 매 라운드 답의 약점·모순 1개를 *판단형 칸 한정*으로 친다. 도전 항목은 프로파일의 challenge bank에서.
5. **되짚기 확정(ⓔ)** — 사용자 말을 *인용*해 "이렇게 이해했는데 맞습니까?"로 확정해야 그 칸 인정.
6. **해석안 제시 = 옵션우선(ⓕ)** — 백지 질문 금지. **모든 질문은 자료에서 뽑은 2~3개 후보를 먼저** 주고 고르거나 고치게 한다. 자유 입력 항상 열림. 후보는 구색이 아니라 진짜 다른 갈래.
   - **예외:** 핵심 메시지 칸은 옵션우선만으로 닫지 않는다(§3 관문 ⓒ 백지 예외).

**한 라운드 = 한 질문.** 가장 약한(🔴/🟡) 하드 칸을 골라 한 번에 하나만 묻는다.

### 라운드 운용 — 한 질문 원칙과 관문의 관계 (테스트 처방)
- **파악 보드 되짚기는 별도 라운드가 아니다.** 보드를 보여주며 "틀린 항목 있으면 먼저 지적해 주세요"라고 한 *같은 턴에* 곧장 핵심 메시지 관문으로 들어간다. 단 사용자가 ✅를 정정하면 반영 후 진행.
- **관문의 〈위계·역의심·백지〉는 한 턴에 묶는다**(위계 명시 + 후보 2~3개 + 백지 질문 = 한 질문). 세 라운드로 쪼개지 않는다.
- **관문 〈둘째 논지〉는 핵심 메시지 확정 *직후* 한 턴.** 즉 핵심 메시지 칸은 보통 **2턴**(위계·역의심·백지 → 확정+둘째 논지)에 닫힌다.
- **관문의 〈역의심〉이 곧 핵심 메시지 칸의 도전이다**(코어 원칙 4 도전과 같은 일). 핵심 메시지 칸에 원칙 4 도전을 *중복* 적용하지 않는다. 원칙 4 도전은 *핵심 메시지 외* 판단형 칸에서 매 라운드 적용.

> ⚠️ 라벨 주의: **코어 6원칙은 ⓐ~ⓕ, 관문 4단계는 이름(위계·역의심·백지·둘째 논지)으로 구분**한다. 같은 동그라미 글자를 두 곳에 쓰지 않는다.

### 압박 대응 — "빨리/그냥 개요만" 요청 (★ 테스트 처방)
시간 압박은 **압축**으로 받지 **생략**으로 받지 않는다.

**압축 경로 = 정확히 3턴. 첫 응답에서 이 3턴을 사용자에게 예고한다:**
1. **핵심 메시지 관문**(§3: 위계·역의심·백지·둘째 논지)
2. **사례·근거 쌍 한 묶음 확인** — 핵심 메시지 확정 *직후*, 어느 사례에 어느 근거를 짝지을지 한 화면으로 확인(구조 골격도 여기서 후보 제시).
3. **개요 산출.**

- **절대 안 건너뛰는 2개 = 1턴과 2턴.** 둘 다 통과 전엔 개요(3턴) 없음.
- **"핵심 메시지만 정하면 바로 개요"는 금지** — 2턴(사례·근거 쌍)을 빠뜨린 *같은 구멍의 약한 변종*이다. 핵심 메시지 확정 직후 "이제 개요"가 아니라 **"이제 사례·근거를 한 번에 맞추고 → 개요"**라고 말한다.
- **압축해도 되는 것:** 칸별 *다회* grill → *후보 묶음 1회 확인*(여러 칸 후보를 한 화면에 모아 한 번에 고르게). 라운드 수는 줄이되 게이트는 유지.
- **즉석 생성 금지:** "나머지는 추정해서 바로 개요" = 생성 금지. 사용자가 명시 승인해도 확정된 중심·사례근거 없이는 산출하지 않고, 추정으로 채운 칸은 🟡·개요에 `[추정]` 표시.

| 압박 핑계 | 실제 |
|----------|------|
| "마감이라 질문은 사치" | 강조점 없는 6쪽이 더 큰 낭비. 압축 경로는 3턴이면 닫힌다. |
| "핵심만 정하면 바로 개요 줘" | 사례·근거 빠진 개요는 검토(§7)에서 터진다. 2턴(사례·근거)을 건너뛰지 않는다. |
| "나머지는 알아서 추정해서 개요로" | 추정 직행 = 즉석 생성. |

## ★ 매 턴 루프 — 측정·판정·분기 (모든 응답에서 강제)

> **이 블록이 "모호도 측정 → 미충족이면 재질문" 루프의 실행체다.** 매 응답은 아래 [A][B][C]를 *이 순서로* 출력한다. 하나라도 건너뛰면 게이트 위반(개요 산출 금지). 핵심 메시지 관문(§3) 단계에선 관문 칸만, grill 단계(§2 [4])부턴 하드 칸 전체를 대상으로 돈다.

**[A] 신호등 보드 — 하드 칸 전체를 매 턴 *다시* 그린다(생략 금지).**

| 칸 | 모호도 | 레벨 | 신호 | 한 줄 근거(인용/빈칸 사유) |
|----|:-----:|:----:|:----:|--------------------------|
| (칸명) | 0.0~1.0 | L0~L3 | 🔴/🟡/🟢 | … |

- **모호도 = 1 − 충족도.** 칸별로 루브릭으로 매긴다: L0=1.0🔴 / L1=0.7🔴 / L2=0.4🟡 / L3=0.0🟢. (숫자는 가시화용, 판정 주체는 신호등.)

**[B] 게이트 판정 — 산술로 못 속인다.**
- 🟢(모호도 0) 아닌 하드 칸이 **1개라도 있으면 → "미통과"**.
- 하드 칸 **전부 🟢 → "통과"**.
- ⚠️ 방향: **모호도 높음(🔴/🟡) = 다시 질문(루프 계속)**, **모호도 0(🟢) = 멈추고 개요로.** (낮을 때 멈춘다 — 거꾸로 아님.)

**[C] 분기 — 판정에 따라 정확히 하나만.**
- **미통과 →** 가장 약한 하드 칸 **하나**(🔴 먼저, 없으면 🟡)를 골라 **옵션우선 질문 1개**. *개요 산출 절대 금지.* → 다음 턴 [A]로 복귀.
- **통과 →** §2 [6] 개요 산출로 진행.
- **무한루프 방지:** 같은 칸이 3턴 넘게 🟢이 안 되면 남은 쟁점을 보고하고 사용자에게 수동 확정 여부를 묻는다(임의 종결 금지, §5).

## 2. 루프 (7단계)

```
[1 덤프]   목적·의도·주제 + 의뢰서 + 자료. 자료는 두 종류로 가른다:
           · 소재 자료(보고서·데이터) = 내용·사실·근거의 원천
           · 레퍼런스(본받을 기사·잘 된 PPT) = 형식·문체·구성의 본보기 (선택)
[2 파악]   자료 정독 → 파악 보드(✅확정/🟡추정/⬜빈칸 + 출처 라벨·신뢰 등급) + (있으면) 레퍼런스 분석 보드 → 사용자에게 보여주고 되짚기
[3 관문]   핵심 메시지(강조점) = 선결 관문. §3 하위 프로토콜로 먼저 확정 (다른 칸 grill 전)
[4 grill]  나머지 하드 칸을 빈칸·약한 순으로 집요하게 되물음(옵션우선·도전·확정). 자료에 이미 있는 건 안 묻는다
[5 게이트] 매 턴 '매 턴 루프'[A][B][C]를 돈다 → 모든 하드 칸 🟢(모호도 0)이면 통과 (칸별 신호등; 단일 임계 가중합 아님)
[6 개요]   섹션별 개요 산출 — 각 섹션: 무슨 주장 + 어떤 사례 + 어떤 근거
[7 검토]   인터뷰어와 분리된 검토자(general-purpose) 디스패치 → 체크리스트 대조 → severity 라우팅
```

### [2] 파악 보드 + provenance
- 두 자료를 읽어 알려진 사실을 ✅/🟡/⬜로 분류. **각 ✅·🟡에 출처 라벨**(`[의뢰서]`/`[소재자료]`/`[레퍼런스]`/`[사용자발화]`/`[AI추정]`)**과 신뢰 등급**(`:확정`/`:웹`/`:추정`)을 단다.
- **`:추정`은 반드시 "제 추정입니다 — 맞습니까?" 확인을 띄운다**(되짚기 전 ✅ 승격 금지). ✅는 출처 행을 한 줄 인용. → 보드가 자료를 *오독해 확정으로 굳히는* 사고를 막는다.
- 레퍼런스가 있으면 **구성·형식·문체** 3축으로 분석(레퍼런스 프로파일). 활용은 **형태 칸(구조·형식·톤)만** 먹이고 **강조점엔 절대 안 닿는다** — 그대로 베끼면 남의 글의 복제.

### [3] 핵심 메시지 관문 — 선결 (★ 가장 중요)
강조점은 글 전체를 떠받치는 중심이다. 자료의 *자기 표제*에 끌려가면 사용자가 이미 버린 프레임에 갇힌다. 닫기 전 아래 네 단계를 *순서대로*(라벨은 이름으로, ⓐ~ⓕ는 코어 원칙 전용):
- **〈위계〉 문서 위계** — 후보 랭킹은 **바인딩 문서(의뢰서·기획 의도) > 원천 자료(보고서·데이터)**. 원천 보고서의 자기 제목·주제어를 1순위로 자동 배정하지 않는다. 의뢰서 방향 ≠ 보고서 방향이면 충돌을 명시("의뢰서는 A, 보고서는 B를 가리킵니다 — 무엇이 중심입니까?").
- **〈역의심〉 프레임 역의심** — "보고서 표제 = 강조점" 가정을 일부러 1회 친다: "이 강조점은 자료가 이미 단 라벨일 뿐, 진짜 하려는 말이 아닐 수 있습니다. 표제를 그대로 쓰면 안 되는 이유는?"
- **〈백지〉 백지 예외** — 자료 후보 2~3개 제시 *직후* 옵션우선을 깨고 백지 질문 1회: **"이 셋 중 어느 것도 아닌, 끝내 남기고 싶은 한 문장은?"** 자료 *너머*를 묻는 유일한 통로.
- **〈둘째 논지〉 둘째 논지 탐지** — 주 강조점이 🟢 후보가 되면 한 번 더: **"이 주 논지를 떠받치거나 반박하는 또 하나의 논지가 있습니까?"** 이대형 글은 흔히 *주 논지 + 둘째 논지*의 이중 구조. 있으면 따로 칸을 만들어 구조에 자리를 준다. **둘째 논지는 1개로 모은다** — 사용자가 둘 이상을 지목하면 한 문장으로 통합하거나 "주 둘째 논지 1 + 보조 1"로 정리해 *되짚기로 확정*(통합본을 인용해 확인). 확정되지 않은 복수 둘째 논지를 들고 다음으로 넘어가지 않는다.
→ 위계·역의심·백지·둘째 논지 + 되짚기 확정(코어 원칙 5) 끝나야 핵심 메시지 🟢. 그 전엔 다른 칸 grill 시작 안 함.

### [4~5] 칸 grill + 채점(신호등)
- 칸 세트·성격(판단형/사실형)·도전 항목은 **프로파일 파일** 참조.
- **루브릭 사다리(괄호=모호도):** L0 빈칸 🔴(1.0) / L1 막연·천장 🔴(0.7) / L2 구체나 미확정 🟡(0.4) / L3 구체+확정+(해당 시)사례·근거+(판단형)도전통과 🟢(0.0). 매 턴 이 값으로 [A] 보드를 채운다.
- **판단형** 칸은 도전, **사실형** 칸은 자료서 확정되면 바로 L3(도전 면제).
- **칸 처리 순서(테스트 처방):** 핵심 메시지·둘째 논지·비목표·독자 확정 → **구조(⑤) 골격 먼저** → 그 골격의 섹션마다 사례(③)·근거(④)를 *쌍으로* 채운다. **사례·근거 쌍은 구조가 정해진 뒤라야 검증된다** — ③·④를 구조 전에 독립 🟢로 올리지 않는다.
  - **레퍼런스가 없으면 구조(⑤)·톤은 빈칸이 아니라 인터뷰어가 후보 제시**: 구조 후보는 *사례의 시간·인과 사슬*에서 뽑고(레퍼런스 대신 사례 내재 순서가 근거), 톤은 의뢰서·매체 규범에서 뽑아 옵션우선으로 묻는다.
- **종료 게이트:** 핵심 메시지 🟢(선결) → 나머지 모든 하드 칸 🟢 → 통과.

### [6] 개요
- 모든 하드 칸 🟢일 때만 산출. 섹션별 {주장·사례·근거}.
- **근거 유형:** 개요 전에 "수치형 vs 서사형" 판정. 근거가 약하면 "채워라"만이 아니라 "(수치)프레임을 버리고 장면·인용으로 가라"도 동등 후보.
- **섹션 순서:** 강조점 랭킹이 아니라 **사례 자체의 시간·인과 사슬**을 따른다.

### [7] 검토 — 분리된 검토자
`general-purpose` 에이전트를 디스패치하되 **인터뷰 산출 개요 + 프로파일 체크리스트**를 주고 검수시킨다(인터뷰어가 자기 산출을 자가채점하지 않게). 검토자 프롬프트에 프로파일 파일의 "§검토 체크리스트"를 그대로 넣는다.
- **severity 라우팅:** 🔴 구조결함(강조점 라벨베낌·요소 누락·사례-근거 깨짐·둘째 논지 누락) → **관문/칸 재오픈**. 🟡 개요결함(순서·비중·중복) → **국소수정** 후 재검토. 🟢 문장씨앗(표현 차원) → **다운스트림 인계**.
- 🔴·🟡 0건이면 개요 확정.
