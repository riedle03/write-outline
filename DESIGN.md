# 이대형 글쓰기 하네스 — 설계 문서 v1

> 작성: 2026-06-27 (v1.5) · 상태: **원고(A)·강의(B) 두 프로파일 + 코어 = 설계 완료, 엔드투엔드 테스트 통과(조건부, 부록 C)·처방 3건 반영** / 인계 스코프·구현 변수 = 미정(§10)
> 목적: 글 또는 강의/PPT **개요**를, *집요한 인터뷰 → 파악 → 개요 → 검토 → 수정* 루프로 정련하는 하네스. 기존 `/deep-interview`가 **얕고 일찍 멈추는** 문제를 정조준해 고친다.

---

## 0. 왜 만드는가 — deep-interview가 얕은 진짜 이유 (코드 확인)

| # | 원인 | 처방 |
|---|------|------|
| 1 | **차원이 소프트웨어용**(Goal/Constraints/Criteria/Entities) — 글에 안 맞음 | 차원을 **글쓰기용**으로 교체(프로파일별) |
| 2 | **채점이 물렁** — 막연한 답도 "답했음"으로 쳐 점수가 떨어짐 | **구체·봉인된 답만 점수↑**(6원칙) |
| 3 | **도전이 늦고 1회**(4·6·8라운드 각 1번) | **매 라운드 도전**(판단형 칸 한정) |
| 4 | **빨리 탈출**(임계 0.2 + 3라운드 조기종료) | **칸별 신호등 게이트**로 정지점 명확화 |

deep-interview의 **모호도-점수 수학과 1질문/라운드·최약 타격 구조는 재사용**하되, 위 4가지를 개조한다.

## 1. 전체 구조

```
[진입 라우터]  "무엇을 쓰십니까?"
   ├─ A: 일반 원고(에세이)
   └─ B: 강의용 PPT 원고
        ↓  (공통) 집요한-인터뷰 코어가 프로파일별 '파악 보드 + 질문 은행'을 끼움
   개요 작성 코어 루프 (칸별 신호등 게이트까지 반복)
        ↓
   확정 개요  → (인계는 보류: 10장 참조)
```

- 공통 재사용물: **집요한-인터뷰 코어(6원칙 + 옵션우선 + 루브릭 채점)**.
- 갈리는 것: **파악 보드의 칸 구성·질문 은행·개요 틀·인계 대상**.

## 2. 개요 작성 코어 루프 (심장)

```
[1 덤프]   목적·의도·주제 + 의뢰서 + 자료를 던짐 — 자료는 두 종류로 가름(§4.1):
           · 소재 자료(연구보고서·데이터) = 내용·사실·근거의 원천
           · 레퍼런스(과월호 기사·잘 된 PPT) = 형식·문체·구성의 본보기
   ↓
[2 파악]   AI가 자료를 정독 → 파악 보드(✅/🟡/⬜ +출처라벨·신뢰등급) + 레퍼런스 분석 보드 제시 → 되짚기 확인
   ↓
[3 되질문] 빈칸·모호한 칸만 집요하게 grill (자료에 이미 있는 건 절대 안 물음)
   ↓
[4 흡수]   답 반영 → 보드 갱신 → 다시 grill  (게이트까지 반복)
   ↓
[5 개요]   모든 하드 칸 🟢이면 개요 초안
   ↓
[6 검토]   인터뷰어와 분리된 검토자가 체크리스트 대조 (§8)
   ↓
[7 수정]   severity별 라우팅 — 🔴 관문 재오픈 / 🟡 국소수정 / 🟢 다운스트림 인계
```

**[2 파악]이 deep-interview와 갈리는 결정적 지점.** deep-interview는 코드를 explore해 "아는 건 안 묻는다"를 하는데, 여기선 그 "코드"가 **사용자가 던진 의뢰서·참고자료**다. 자료를 먼저 읽어 빈칸만 골라낸다.

## 3. 집요한-인터뷰 코어 — 6원칙 (매 라운드 강제)

deep-interview가 *안 하던* 것들:

1. **막연답 거부** — 추상 답이면 점수 안 주고 좁히는 꼬리질문. (ⓐ)
2. **3겹 파기** — 한 답에 "왜 → 구체적으로(숫자·고유명사·장면) → 그래서 뭐가 달라지나" 최소 3단. (ⓑ)
3. **게이트 전 산출 금지** — 모든 하드 칸이 통과선 넘기 전엔 개요 안 씀. (ⓒ)
4. **도전 모드 상시** — 매 라운드 답의 약점·모순 1개 짚음(*판단형 칸 한정*). (ⓓ)
5. **되짚기 봉인** — 사용자 말을 *인용*해 "이렇게 이해했는데 맞나요?"로 확정해야 그 칸 인정. (ⓔ)
6. **해석안 제시** — 백지 질문 금지. (ⓕ → R-옵션우선으로 승격)

### R-옵션우선 (기본 규칙)
모든 질문은 **선택지를 먼저** 준다. AI가 *자료에서 뽑은* 2~3개 후보 제시 → 사용자는 고르거나 고침. 자유 입력("다른 표현으로") 항상 열림. 후보는 들러리가 아니라 *진짜 다른 갈래*여야 하고 유도·편향 금지.
> 예: "핵심 메시지가 뭐예요?"(❌ 백지) → "자료를 보니 ①통합 ②끝까지 ③AI시대 중 하나 같습니다. 어느 쪽? 아니면 다르게?"(⭕)

> ⚠️ **예외 — 핵심 메시지 칸은 옵션우선만으로 닫지 않는다.** 백테스트(부록 B)에서 옵션우선이 *자료에 적힌 프레임*에 강조점을 가두는 천장이 확인됐다. 진짜 강조점은 자료 어디에도 그 각도로 안 적힐 수 있다. → **핵심 메시지 관문 하위 프로토콜(§5.1)을 따른다.**

### 도전 모드(ⓓ) 탄알 — P1~P16 배선 (★ 형식적 도전 방지)

도전 모드는 "정말 그럴까요?" 연극이 되면 죽는다. 실탄은 `revision_pattern.md`(P1~P16)와 공적 표현 원칙 — *이대형 본인이 반복적으로 틀리는 지점의 카탈로그*다. **판단형 칸마다** 아래 탄알을 1발 이상 장전한다(중 하나라도 걸리면 그 칸 🟢 보류).

| 칸 | 도전 탄알(출처) |
|----|----------------|
| 핵심 메시지 | "자료가 이미 단 라벨을 그대로 베낀 것 아닌가?"(§5.1ⓑ·잔여결함) · "통념 세워 꺾는 설교 틀인가?"(P1) |
| 독자·목적 | "학생·집단을 단정·일반화('학생들은 모두…')하지 않았나?"(P2) · "구조적 어려움을 미화하지 않았나?"(P3) |
| 중심 사례 | "추상이 아니라 실제 이름·맥락(브리스크티칭 대화·졸업 후 취업)이 박혔나?"(P15) · "은유로 때운 장면 아닌가?"(P13) |
| 구조 | "훈계조 대비 구조가 골격에 박혔나?"(P1) · "설계를 해설·예고하는 자기지칭('첫 번째 결정이었다')이 섞였나?"(P12·P8) |
| 톤 | "필자 실천은 단언하되 학생·사회 진단은 신중한가(자신감 비대칭)?"(P2) |

> 문장 차원 탄알(P9 의인화·P10 시제·P11 한자투·P14 압축·P16 영문병기)은 개요가 아니라 **다운스트림(passage-revision-loop·humanize-korean)** 몫 → 검토 단계(§8) 체크리스트로 넘긴다. 도전 모드는 *개요에 박히는* 구조·진단 차원(P1·P2·P3·P8·P12·P13·P15)만 친다.

## 4. 파악 보드 — grill 전에 먼저 펼친다

자료 흡수 직후 AI가 이 표를 **보여주고** → 되짚기(ⓔ)로 확정·수정받은 뒤 → 빈칸부터 grill.

| 구분 | 뜻 | 예(서울교육 원고였다면) |
|------|----|------------------------|
| ✅ **확정** | 자료에서 직접 읽힘 | 코너명·분량·마감·주제 범위·평어체 규정 |
| 🟡 **추정** | 유추했으나 확인 필요 | 핵심 메시지 후보·독자상·톤 |
| ⬜ **빈칸** | 자료에 없음 → 반드시 grill | "끝내 남길 한 문장"·중심 사례·구조 |

→ 🟡추정·자료 후보가 **그대로 옵션우선 질문의 선택지**가 된다.

**G2 — 출처 라벨 + 신뢰 등급 (k-teacher provenance 차용, ★ 보드 오독 차단).**
모든 ✅·🟡 항목에 *어디서 왔는지*와 *얼마나 믿는지*를 함께 단다. 보드가 자료를 **잘못 읽어 ✅로 굳히는** 사고(아키텍트 H2)를 막는 장치다.
- **출처 라벨:** `[의뢰서]` / `[소재자료]` / `[레퍼런스]` / `[사용자발화]` / `[AI추정]`
- **신뢰 등급:** `:확정`(자료에 명시) / `:웹`(외부 출처로 확인) / `:추정`(AI 유추)
- **강제 규칙:** `:추정` 항목은 **반드시** "이건 제 추정입니다 — 맞습니까?" 확인을 띄운다(되짚기 전 ✅ 승격 금지). ✅ 항목은 출처 행을 한 줄 인용한다.
- §5.1 ⓐ 문서 위계와 정합: 같은 사실이 `[의뢰서]`와 `[소재자료]`에서 충돌하면 의뢰서를 우선하고 충돌을 명시한다.

### 4.1 레퍼런스 분석·활용 보드 (★ 신규 — 사용자 요청)

> **왜 따로 두나:** 글·PPT를 쓸 때 **본받을 본보기**(과월호 기사, 잘 만든 PPT)가 있으면 그 *형식·문체·구성*을 분석해 활용한다. 단 **소재 자료와 성격이 다르다** — 소재는 *내용*을, 레퍼런스는 *형태*를 준다. 둘을 섞으면 안 된다. 기존 자산 `reference-scout`·`reference-matching` 재사용.

**분석 — 레퍼런스에서 뽑는 3축(레퍼런스 프로파일):**
| 축 | 뽑을 것 |
|----|---------|
| **구성** | 섹션 흐름·도입 방식·마무리 패턴·논지 전개 골격 |
| **형식** | 분량 배분·사진/표 위치와 빈도·소제목 스타일·문단 길이 |
| **문체** | 호흡·평어체/경어체·톤·문장 길이·어휘 결 |

**활용 — 베끼지 말고 변용:**
- 레퍼런스는 **구조·형식·톤 칸만** 먹인다. **핵심 메시지(강조점)는 절대 먹이지 않는다** — 백테스트에서 *보고서 라벨*이 강조점을 납치한 것과 같은 위험. 레퍼런스를 그대로 베끼면 "남의 글의 복제"가 된다.
- DNA만 추출해 **내 강조점에 맞춰 변용.** "이 레퍼런스는 *왜* 그렇게 했나"를 묻고, 내 글에 맞으면 차용·아니면 의도적 이탈.

**대조 — 검토 단계(§8)에서:**
- "레퍼런스의 강점을 살렸나?" / "의도적으로 벗어난 지점은 정당한가(내 강조점 때문인가, 그냥 놓친 건가)?"
- 레퍼런스가 없으면 이 보드는 건너뛴다(선택 입력).

## 5. 프로파일 A(원고) — 칸(차원) 구성

| # | 칸 | 성격 | 게이트 | 레퍼런스 먹임 |
|---|----|------|--------|:---:|
| 1 | **핵심 메시지(강조점)** — 끝내 남길 한 문장 | 판단형 | **선결 관문** | ✗(절대) |
| 1b | **비목표(non-goals)** — 이 글이 *다루지 않을·주장하지 않을* 것 | 판단형 | **하드(★G3)** | ✗ |
| 2 | **독자·목적** — 누구에게, 읽고 뭘 바꾸나 | 판단형 | 하드 | ✗ |
| 3 | **중심 사례·일화** — 구체적 장면·인물·수치 | 판단형 | 하드 | ✗ |
| 4 | **근거·전거** — 사례를 받치는 학술/데이터 근거의 *종류·방향* | 사실형 | 하드 | ✗ |
| 5 | **구조** — 도입–전개–마무리 골격 | 판단형 | 하드 | ✅ |
| 6 | **형식·제약** — 분량·코너·마감·금기 | 사실형 | 하드(주로 ✅자동확정) | ✅ |
| — | **톤** | 판단형 | 소프트(산출 품질) | ✅ |
| — | **제목·후크** | — | **산출물**(강조점 뒤 도출) | △ |

> **G3 비목표(1b):** k-teacher의 mandatory hard gate "what student will NOT learn"의 글쓰기 판. 6쪽 제약상 *무엇을 뺄지*가 강조점만큼 글을 결정한다. 핵심 메시지(1) 🟢 직후 묻는다("이 강조점을 위해 *버릴* 곁가지는?"). 비목표가 흐릿하면 구조(5)가 비대해진다.
> **레퍼런스 먹임 열:** 레퍼런스는 형태 칸(구조·형식·톤)만 먹인다. 판단 칸(메시지·비목표·독자·사례)에는 절대 안 닿는다(§4.1).

**DNA: 사례(3)와 근거(4)는 *쌍*으로 잡는다.** 이대형 글의 핵심. (개요에서 근거를 안 잡으면 검토 단계에서 매번 터짐 — 실제 경험.)

> **★ 근거 유형 게이트 (4번 칸, 테스트 처방).** 개요 확정 전 **"이 글은 수치형인가 서사형인가"**를 먼저 판정한다. 테스트(부록 C)에서 하네스는 보고서에 효과크기가 있으니 *기본값으로 수치 귀결*을 깔았는데, 실제 최종본은 효과크기를 **하나도 안 쓰고** 전부 장면·인용으로 갔다 — 근거 열 전면 오답. **레퍼런스(서울교육 과월호=서사형)가 이 유형을 잡아준다.** 근거 칸이 빈칸·약함이면 "**채워라**"만이 아니라 "**(수치) 프레임을 버리고 장면·인용으로 가라**"를 *동등한 후보*로 띄운다.
> **★ 구조 정렬 기준 (5번 칸, 테스트 처방).** 개요 섹션 배열은 *내가 정한 강조점 랭킹*이 아니라 **사례 자체의 시간·인과 사슬**을 따른다. 테스트에서 하네스가 깊이읽기를 척추로 삼아 국어 단원을 앞당겨, 저자의 '정서→국어→AI' 인과 계단(마음이 열려야 긴 글을 읽는다)을 깼다. 사례에 내재한 순서가 있으면 그것이 우선.

### 5.1 핵심 메시지 관문 — 하위 프로토콜 (백테스트 처방, ★)

> **왜 이것만 따로 두나:** 강조점은 척추다. 구조·사례 배치·제목·톤이 전부 여기 매달린다. 그런데 백테스트(부록 B)에서 코어 v1은 자료(연구보고서)의 *자기 표제*에 강조점을 끌려가, 사용자가 **이미 버린 프레임**(주도성·문해력3.0)에 봉인됐다. 정작 정답 강조점("따로 길러선 안 될 문해력, 하나로")은 **두 자료 어디에도 그 각도로 안 적혀 있었다.** 이 관문은 그 세 구멍을 막는다.

이 칸을 🟢으로 닫기 전, 아래 3단을 *순서대로* 강제한다.

**ⓐ 문서 위계 — 강조점 후보 랭킹은 의뢰서가 먼저.**
강조점 후보를 자료에서 뽑되, **바인딩 문서(의뢰서·기획 의도) > 원천 자료(연구보고서·데이터)** 순으로 가중한다. 원천 보고서의 *자기 제목·주제어*를 1순위 후보로 자동 배정하지 않는다. 의뢰서가 가리키는 방향과 보고서가 미는 방향이 **다르면** 그 충돌 자체를 사용자에게 명시("의뢰서는 A를, 보고서는 B를 가리킵니다 — 무엇이 척추입니까?").

**ⓑ 프레임 역의심 — 통과 전 1회 의무 도전.**
"보고서 제목/주제어 = 이 글의 강조점"이라는 가정을 *일부러* 친다. 최소 1회: "이 강조점은 자료가 이미 단 라벨일 뿐, 당신이 진짜 하려는 말이 아닐 수 있습니다. 보고서 표제를 그대로 쓰면 안 되는 이유가 있습니까?" 이 도전을 버티거나, 버티지 못해 후보를 바꿔야 🟢 후보 자격이 생긴다.

**ⓒ 백지 예외 — 옵션우선을 한 번 깬다.**
자료 후보 2~3개를 제시한 *직후*, 반드시 백지 질문을 1회 연다: **"이 셋 중 어느 것도 아닌, 당신이 끝내 남기고 싶은 한 문장은 무엇입니까?"** 핵심 메시지 칸에서만 옵션우선의 예외 — 자료 *너머*를 묻는다. 자료에 없는 강조점이 떠오를 유일한 통로다.

**ⓓ 역기둥 탐지 — 척추를 하나만 찾고 끝내지 않는다 (★ 테스트 처방).**
주 강조점이 🟢 후보가 되면, 닫기 전에 한 번 더 묻는다: **"이 주 논지를 *떠받치거나 반박하는* 둘째 기둥이 있습니까?"** 엔드투엔드 테스트(부록 C)에서 하네스는 통합(주 논지)만 잡고 최종본의 둘째 기둥 "**수업 설계만으로는 부족하다 — 교사의 개별지도**"(P2 자신감 비대칭과 직결)를 통째로 놓쳤다. 이대형 글은 흔히 *주 논지 + 역기둥*의 이중 구조다. 역기둥이 있으면 별도 칸으로 박아 구조(5)에 자리를 준다.

→ ⓐⓑⓒⓓ 통과 + 되짚기 봉인(ⓔ)까지 끝나야 핵심 메시지 🟢. 그 전엔 다른 칸 grill 시작 안 함(선결 관문, §7).

## 6. 모호도 채점 — 루브릭 사다리 + 칸별 신호등

### 6.1 칸마다 루브릭 사다리 (vibe 0~1 ❌, 레벨 ⭕)

| 레벨 | 신호등 | 뜻 | 통과 |
|------|--------|----|----|
| **L0** 0.0 | 🔴 | 빈칸·미응답 | ✗ |
| **L1** 0.3 | 🔴 | **막연·일반론**("교사에게 도움 되게") | ✗(천장) |
| **L2** 0.6 | 🟡 | **구체적이나 미봉인**(숫자·고유명사·장면 있으나 되짚기 전/근거 없음) | △ |
| **L3** 1.0 | 🟢 | **구체 + 봉인 + (해당 시) 사례·근거 결합 + (판단형) 도전 통과** | ✓ |

- L1→L2: 답에 *숫자·고유명사·장면·행동*이 들어옴.
- L2→L3: ⓔ인용 확인 + (판단형) ⓓ도전 버팀 + (근거 칸) 전거 확보.

### 6.2 4개 이빨 — 점수를 *안* 올리는 규칙

1. 추상어·일반론 → **L1 천장**
2. "알아서 해줘"/후보 미선택 → **L2 못 넘음**(미봉인)
3. 자료 근거 없는 주장 → **근거·전거 칸 L2 천장**
4. 도전 모드가 모순 발견 → 그 칸 **L1로 강등(재오픈)** + 연결 칸(메시지↔사례↔구조)도 재점검

### 6.3 채점에 반영한 4개 수정 (v1 핵심)

1. **판단형 vs 사실형 차등** — 사실형(형식·제약, 근거의 서지정보)은 자료서 확정되면 **바로 L3, 도전 불필요.** 도전은 *판단*에만.
2. **단일 모호도 숫자 → 칸별 신호등(🔴🟡🟢)** — 가중합 소수점(가짜 정밀) 폐기. *어느 칸이 막혔는지* 한눈에.
3. **핵심 메시지 = 선결 관문**(가중치 아님) — 핵심 메시지 🟢 전엔 다른 칸 grill 시작 안 함(곱셈 관계 · "강조점 먼저").
4. **L3 기준 = 개요 적정 수준** — 개요에선 근거의 *종류·방향*까지만(정밀 서지·페이지는 다운스트림). 개요가 논문 검증이 되지 않게.

## 8. 검토→수정 단계 (개요 산출 후, ★)

게이트를 통과해 개요 초안이 나오면, **인터뷰어와 분리된 검토자**가 개요를 체크리스트에 대조한다(자기채점 구멍 차단 — 🟢을 준 주체가 자기 산출을 검수하지 않는다). 결함은 severity로 분류해 *grill 재진입* 또는 *개요 국소수정*으로 보낸다.

### 8.1 검토 체크리스트 — 3부

**(가) 내용 정합 — 개요가 자료·의뢰를 덮는가**
- ☐ 의뢰서 요소(예: AI시대·긴 글 리터러시·최소 성취수준 보장·탐구수업 사례)를 개요가 모두 덮는가
- ☐ 강조점(핵심 메시지)이 모든 섹션을 꿰는 **척추**인가, 아니면 한 단락 장식인가
- ☐ **역기둥(§5.1ⓓ) 자리 있나** — 주 논지를 떠받치거나 반박하는 둘째 기둥이 개요에 섹션으로 들어왔나(이중 구조 누락 점검)
- ☐ **비목표(1b) 준수** — 버리기로 한 곁가지가 개요에 슬그머니 돌아오지 않았나
- ☐ **섹션 × {사례, 근거} 매트릭스** — 각 주장 섹션이 사례·근거를 *쌍으로* 가졌나(칸3·4 독립 🟢로는 불충분 — DNA 강제 게이트)
- ☐ **근거 유형 정합** — 수치형/서사형 판정과 실제 근거 열이 맞나(서사형인데 효과크기 깔지 않았나 — 부록 C 오답 방지)
- ☐ **섹션 순서 = 사례 인과/시간 순**인가 — 강조점 랭킹으로 순서를 뒤집지 않았나
- ☐ 구조가 도입→전개→마무리로 *진행*하는가(나열이 아니라)

**(나) 라벨 vs 구조역전 — 잔여결함 게이트 (★ 부록 B)**
- ☐ 강조점·소제목이 **자료가 이미 단 라벨**(주도성·문해력3.0·"통합적 문해력")을 그대로 베꼈는가, 아니면 자료의 구조를 **뒤집은 새 각도**(예: 보고서의 3분할을 '한 번의 읽기 경험'으로 재서술)인가
- → 라벨 그대로면 FAIL → 핵심 메시지 관문(§5.1) **재오픈**. 이 게이트가 §5.1이 못 막은 마지막 한 걸음을 잡는다.

**(다) 공적 표현·이대형 DNA — P1~P16 개요 차원 (★ 배선)**
- ☐ P1 훈계조 대비 구조("손쉬운 처방은 ~, 그러나")가 골격에 박혔나
- ☐ P2 자신감 비대칭 — 필자 실천 단언 / 학생·사회 진단 신중
- ☐ P3 구조적 어려움(누적 결손·단기간 변화 곤란)을 솔직히 인정하되 학생 탓 아님
- ☐ P8·P12 메타텍스트·설계 해설("이것이 첫 번째 결정이었다")이 개요 골격에 없나
- ☐ P13 은유로 때운 섹션 제목·논지 없나
- ☐ P15 사례가 추상 아닌 실제 이름·맥락으로 잡혔나
- ☐ P6 인접 섹션 간 같은 진단·어휘 중복 없나

**(라) 레퍼런스 대조 — 본보기가 있을 때만 (★§4.1)**
- ☐ 레퍼런스 프로파일의 **구성·형식·문체 강점**을 개요가 살렸나
- ☐ 레퍼런스에서 **의도적으로 벗어난** 지점이 있다면, 내 강조점 때문인가(정당) 아니면 그냥 놓친 것인가(결함)
- ☐ 레퍼런스를 **그대로 베껴** 강조점·논지가 남의 글의 복제가 되지 않았나(라벨 베낌과 같은 위험)

> **문장 차원(P4·P9·P10·P11·P14·P16)은 검토 대상이되 수정은 다운스트림으로 인계** — 개요 단계에선 "이 결함이 *개요 결정*인가 *문장 윤문*인가"만 판정해 라우팅한다.

### 8.2 수정 라우팅 — severity별

| severity | 정의 | 처리 |
|----------|------|------|
| 🔴 구조결함 | 강조점 라벨베낌(나), 4요소 누락, 사례-근거 쌍 깨짐, 훈계조 골격(P1) | **관문/칸 재오픈** → 해당 칸 grill 재개 |
| 🟡 개요결함 | 섹션 순서·비중, 중복(P6), 설계 메타(P12) | 개요 내 **국소수정** 후 재검토 |
| 🟢 문장씨앗 | P9·P10·P11·P14·P16 등 표현 차원 | 개요엔 메모만, **다운스트림(passage-revision-loop·humanize-korean)** 인계 |

### 8.3 루프 종료
검토에서 🔴 0건 + 🟡 0건이면 개요 확정. 🔴이 관문 재오픈을 부르면 §5.1·해당 칸으로 돌아가 다시 게이트까지. (최대 회차는 `[당신이 정할 변수]`.)

## 9. 프로파일 B(강의/PPT) — 칸 정련 (★)

> **코어는 그대로:** 6원칙·옵션우선·루브릭 L0~L3·신호등·provenance(§4 G2)·레퍼런스 보드(§4.1)·검토→수정(§8). **칸 세트와 고유 게이트만 갈아 끼운다.**
>
> **글과 다른 점(당신 지적 "흐름이 다르다"):** 글은 *혼자 읽는 선형*, 강의는 *청중 앞 실시간·모듈 단위·분(分) 제약·1슬라이드=1메시지*. 그래서 강의엔 **시간·인지부하라는 물리 제약**이 추가되고, 척추는 "한 줄 약속"(청중이 가져갈 단 하나)이다.

### 9.1 칸(차원) 구성

| # | 칸 | 성격 | 게이트 | 레퍼런스 먹임 |
|---|----|------|--------|:---:|
| 1 | **한 줄 약속(강조점)** — 청중이 끝나고 가져갈 *단 하나* | 판단형 | **선결 관문** | ✗(절대) |
| 1b | **비목표** — 시간상 *안 다룰* 것(쳐낼 곁가지) | 판단형 | 하드 | ✗ |
| 2 | **청중·맥락** — 누구(직급·교과·사전지식)·규모·자발성·**총 시간** | 사실형 | 하드(주로 ✅) | ✗ |
| 3 | **모듈별 takeaway** — 각 모듈서 청중이 *할 수 있게/믿게* 될 것 **+ 그 증거**(뭘 하면 도달) | 판단형 | 하드 | ✗ |
| 4 | **중심 사례·근거** — 라이브 시연/스토리 + 받치는 근거(글과 같은 **쌍 DNA**) | 판단/사실 | 하드 | ✗ |
| 5 | **모듈·시간 설계** — 세션 분할 + **분 단위 시간 배분**(도입–전개–실습–마무리) | 판단형 | 하드 | ✅ |
| 6 | **상호작용·실습 설계** — 청중 참여 장치(질문·토의·실습·투표)와 배치 | 판단형 | 하드 | ✅ |
| 7 | **전달·슬라이드 설계** — **1슬라이드=1메시지**, *말할 것 vs 띄울 것* 분리, 시각화 | 사실/판단 | 하드 | ✅ |
| — | **톤·에너지** — 청중 각성 관리(완급) | 판단형 | 소프트 | ✅ |
| — | **제목·오프닝 후크** | — | 산출물(약속 뒤 도출) | △ |

### 9.2 강의 고유 하드 게이트 2개 (글엔 없음)

- **⏱ 시간 예산 게이트** — 모듈 시간 합 = 총 시간(±버퍼). **산술이라 거짓 못 함**(hwpx 줄 계산과 같은 성격). 초과면 통과 불가 → **비목표(1b)로 쳐내** 줄인다. "다 넣고 싶다"를 막는 물리 한계.
- **🪧 1슬라이드=1메시지 게이트** — 한 슬라이드에 메시지 2개 이상이면 FAIL → 쪼갠다. *읽는 슬라이드 = 죽은 슬라이드*. 글의 압축(P14)이 강의에선 슬라이드 게이트로.

### 9.3 강의 고유 도전 탄알 (§3 도전 모드 + 강의 특유)

| 칸 | 탄알 |
|----|------|
| 한 줄 약속 | "정보 *나열*인가, 청중이 *가져갈* 하나인가?"(전달≠takeaway) |
| 모듈 takeaway | "'이해한다'류 모호어 아닌가 — 청중이 끝나고 *뭘 할 수 있나*로 말했나?"(assessment-first) |
| 모듈·시간 | "이 모듈 실제 분 단위로 돌아가나, 욱여넣었나?"(⏱게이트) |
| 상호작용 | "청중이 *졸* 지점은? 30분+ 일방 전달 구간 없나?" |
| 슬라이드 | "이 슬라이드 글이 너무 많지 않나?"(🪧게이트·P14) |
| 사례·톤 | 실제 이름·맥락(P15)·은유 절제(P13)·자신감 비대칭(P2)·훈계조 금지(P1) — 글과 공유 |

### 9.4 검토 단계(§8) 강의판 차이

§8 체크리스트를 쓰되 **(가)에 강의 항목 추가**:
- ☐ 한 줄 약속이 *모든 모듈*을 꿰는 척추인가
- ☐ 모듈별 takeaway가 *증거*로 정의됐나(모호어 아님)
- ☐ **⏱ 시간 예산** 맞나 / **🪧 1슬라이드=1메시지** 지켜졌나
- ☐ 청중 *졸 구간*(상호작용 공백 30분+) 없나
- ☐ *말할 것 vs 띄울 것* 분리됐나(슬라이드가 대본이 아님)

→ severity 라우팅 동일. 슬라이드 *문구* 다듬기는 다운스트림(`leedaehyung-pptx`·`lecture-redesign-harness`)으로 인계.

## 부록 B. 백테스트 기록 — §5.1 하위 프로토콜의 근거 (2026-06-27)

**방법:** 콜드 시뮬레이터(정답 미열람)에게 코어 v1의 파악 보드+6칸+게이트를 **의뢰서·연구보고서에만** 적용시켜, 도달하는 강조점을 산출. 정답지(최종본 「따로 길러선 안 될 문해력, 하나로 키우다」)는 시뮬레이터에 숨김.

**결과:** 코어 v1이 도달한 강조점 = **"문해력 3.0 기반 탐구수업으로 최소성취 보장하며 주도성을 기른다."**
→ **정답("주도성도 3.0도 아닌, 따로 길러선 안 될 문해력을 하나로")과 반대 방향.** 사용자가 명시적으로 버린 두 프레임에 정확히 봉인됨.

**드러난 인과 3건 → §5.1 처방과 1:1 대응:**

| 결함 | 증거 | 처방 |
|------|------|------|
| 문서 위계가 거꾸로 | v1 옵션우선이 보고서 자기표제(주도성)를 4중 실탄으로 1순위 배정. 의뢰서('문해력', 주도성 0회)는 L2로 밀림 | §5.1 ⓐ 문서 위계 |
| 도전이 프레임을 못 침 | ⓓ가 하위 주장만 침. 프레임을 칠 무기(의뢰서)가 v1 규칙상 자동 발화 안 됨 | §5.1 ⓑ 프레임 역의심 |
| 옵션우선의 천장 | 정답 "통합/하나로"는 두 자료 어디에도 그 각도로 안 적힘 → 자료 후보만 제시하는 옵션우선이 구조적으로 못 띄움 | §5.1 ⓒ 백지 예외 |

**재검증(§5.1 적용 후, 콜드):** 판정 = **부분적으로 열림(열림 쪽).**
- ⓐ가 실효: 의뢰서에 "주도성" 부재 → 보고서 자기표제(주도성·문해력3.0)가 **1순위→3·4순위 강등**, 의뢰서발 "긴 글 심층 읽기 문해력"이 1순위 부상.
- ⓑ가 정확히 작동: "주도성·문해력3.0 = 대회 출품용 포장 라벨"임을 폭로(의뢰서는 "보고서 아닌 에세이·생생한 사례" 요구).
- ⓒ 백지가 *구조적으로* "세 문해력을 따로 안 기르고 한 번의 읽기 경험으로 통합" 문장을 강제 산출 → 자료-너머 통로 **열림 확인.**
- **잔여 결함(정직):** "통합"이라는 *단어*는 보고서에 이미 라벨로 있음(용어정의 "유기적 통합"). 관문은 *얕은 라벨 채택*("통합적 문해력")과 *깊은 구조 역전*(3분할→한 번의 읽기 경험)을 **구분 못 함.** 그 마지막 한 걸음은 집필자 판단에 남음. → **검토 단계(§8.1-나 '라벨 vs 구조역전 게이트')가 잡도록 설계 완료(v1.2).**
